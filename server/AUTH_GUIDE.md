# 浏览器数据同步 - 认证系统使用指南

## 功能概述

本系统已添加完整的用户认证功能，包括：
- ✅ 首次启动自动生成管理员账号
- ✅ 登录界面
- ✅ 密码修改功能
- ✅ 会话管理（基于 localStorage）
- ✅ API 接口认证保护

## 首次使用

### 1. 启动服务器

启动服务器后，会在控制台显示自动生成的管理员账号：

```
========================================
  ADMIN USER CREATED
  Username: admin
  Password: <随机生成的密码>
  PLEASE SAVE THIS PASSWORD!
========================================
```

**重要：** 请立即保存此密码，后续登录需要使用！

### 2. 访问管理面板

打开浏览器访问：`http://localhost:8080/`

系统会自动跳转到登录页面 `http://localhost:8080/login.html`

### 3. 登录

使用控制台显示的账号密码登录：
- **用户名**: admin
- **密码**: 服务器启动时显示的随机密码

如果是首次登录，系统会显示初始账号信息提示框。

## 修改密码

登录成功后：

1. 点击左侧导航栏的 **设置**
2. 在"修改密码"部分：
   - 输入当前密码
   - 输入新密码（至少 6 位）
   - 确认新密码
3. 点击 **修改密码** 按钮
4. 修改成功后会自动退出，需要使用新密码重新登录

## 退出登录

在设置页面点击 **退出登录** 按钮即可。

## 安全特性

### 1. API 保护
所有数据相关的 API 接口都需要认证：
- `/api/status` - 查看状态
- `/api/bookmarks` - 书签管理
- `/api/history` - 历史记录
- `/api/cookies` - Cookie 管理
- `/api/data` - 删除数据
- `/api/deduplicate` - 去重功能

### 2. 公开接口
以下接口不需要认证：
- `/api/auth/login` - 登录
- `/api/auth/check` - 检查登录状态
- `/api/auth/initial` - 获取初始账号信息

### 3. 会话管理
- 登录成功后，token 保存在 localStorage
- 刷新页面自动验证登录状态
- 未登录会自动跳转到登录页

### 4. 密码加密
- 密码使用 SHA-256 哈希存储
- 服务器只保存密码哈希值
- 不存储明文密码

## 数据存储

用户账号信息保存在：`server/src/auth.json`

文件格式：
```json
{
  "admin": {
    "username": "admin",
    "passwordHash": "加密后的密码",
    "createdAt": "创建时间"
  }
}
```

## Docker 部署

使用 Docker 部署时，账号信息会保存在容器内。如果需要持久化：

1. 挂载数据卷：
```yaml
volumes:
  - ./auth-data:/app/src
```

2. 或者使用 docker-compose（推荐）：
```yaml
version: '3'
services:
  browser-sync:
    image: browser-sync:latest
    ports:
      - "8080:8080"
    volumes:
      - ./auth-data:/app/src
```

## 注意事项

1. **首次启动密码**：务必保存服务器启动时显示的密码
2. **密码修改**：修改密码后需要重新登录
3. **忘记密码**：删除 `server/src/auth.json` 文件，重启服务器会重新生成 admin 账号
4. **会话过期**：目前会话永久有效，除非手动退出或删除 localStorage

## 故障排除

### 问题：无法登录
- 检查用户名和密码是否正确
- 确认服务器已启动
- 查看浏览器控制台是否有错误

### 问题：登录后自动退出
- 检查 localStorage 是否被清除
- 确认没有使用隐私浏览模式
- 检查浏览器是否允许保存本地数据

### 问题：修改密码失败
- 确认旧密码正确
- 新密码长度至少 6 位
- 两次输入的新密码必须一致
